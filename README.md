## Реализация методов

### 1. Разреженный оптический поток (Lucas-Kanade)
Метод отслеживает перемещение отдельных точек (углов) между кадрами. Используется для построения траекторий.

*   **Детекция точек:** Алгоритм Shi-Tomasi (`goodFeaturesToTrack`).
    *   `maxCorners=100`, `qualityLevel=0.1`, `minDistance=7`.
*   **Параметры потока (LK):**
    *   `winSize=(15, 15)` — размер окна поиска.
    *   `maxLevel=2` — использование пирамид для отслеживания больших смещений.
    *   `criteria` — остановка по 10 итерациям или точности 0.03.

<img width="1463" height="807" alt="image" src="https://github.com/user-attachments/assets/1d3e8689-5922-413b-8de3-0742b26a8745" />

**Результат визуализации:**
На полученном итоговом изображении отчетливо видны траектории движения людей:
*   **Направление**: Разноцветные линии показывают вектор движения каждого человека.
*   **Характер движения**: Траектории плавные, что говорит о низком уровне шума и стабильной работе алгоритма calcOpticalFlowPyrLK. Длина линии коррелирует со скоростью объекта и временем его нахождения в кадре.
*   Текущее положение людей обозначено точкой


### 2. Плотный оптический поток (Farnebäck)
Метод вычисляет вектор смещения для каждого пикселя в кадре. Используется для сегментации и анализа кадра целиком.

*   **Параметры:** `pyr_scale=0.5`, `levels=3`, `winsize=15`, `iterations=3`, `poly_n=5`, `poly_sigma=1.2`.
*   **Постобработка:** Пороговое отсечение (`threshold = 2.0`) для получения бинарной маски.

<img width="1902" height="677" alt="image" src="https://github.com/user-attachments/assets/6250fac0-41a9-4d69-80ce-0406214da522" />

**Результат визуализации:**
- На левой части итогового видео изображено поле движения (HSV визуализация), которое наглядно показывает:

*   **Направление потоков**. На изображении хорошо видны направление двух групп движущихся людей (бирюзово-голубые и красно-оранжевые). Метод успешно определил разные направления движения людей.
*   **Скорость движения**. Яркость цветных пятен варьируется, что говорит о разной скорости перемещения отдельных людей.
*   **Недвижущиеся объекты** (фон, асфальт) остаются полностью черным. Алгоритм корректно вычисляет нулевое смещение для неподвижных объектов

- На правой части итогового видео изображена бинарная маска, которая показывает:

*   **Сегментацию объектов**. Белые области соответствуют движущимся людям. Маска отделяет движущиеся объекты от фона.
*   **Форму объектов**. Силуэты на маске соответствуют проекции людей (вид сверху)

## Сравнительный анализ чувствительности

Был проведен эксперимент с искусственным ухудшением качества видео по 4 категориям:
1.  **Texture** (Пониженный контраст).
2.  **Motion Blur** (Размытие по Гауссу).
3.  **Shadows** (Имитация теней/темноты).
4.  **Noise** (Гауссовский шум).

### Метрики
*   **LK:** *Lost Tracks Ratio* (Доля потерянных треков).
*   **Farnebäck:** *Active Pixels* (Доля площади кадра, классифицированная как движение).

## Разбор ошибок

### Ошибки Lucas-Kanade:
*   **Потеря точек в тенях:** Метод зависим от наличия острых углов. В зонах с низким контрастом (тени) алгоритм перестает считать область точкой и сбрасывает трек.

### Ошибки Farnebäck?
*   **Неустойчив к шуму: Хаотичное изменение яркости пикселей (шум) интерпретируется алгоритмом как быстрое движение. Без предварительного сглаживания (Gaussian Blur) метод неприменим к видео с шумом.

## Выбор метода для реальных задач


| Критерий | Выбираем Lucas-Kanade (LK) | Выбираем Farnebäck (Dense) |
| :--- | :--- | :--- |
| **Цель** | Получить **траекторию** $(x, y)$ и скорость конкретного объекта. | Получить **бинарную маску**, выделить силуэт, оценить плотность потока. |
| **Искажения видео** | **Шумное видео**: LK лучше фильтрует шум. | **Размытое видео** (Motion Blur): Dense лучше видит "пятна" без четких границ. |
| **Ресурсы** |  Высокая скорость | Ресурсоемкий алгоритм. |

**Вывод:**
*   Для задач навигации, стабилизации и трекинга отдельных объектов — **Lucas-Kanade**.
*   Для задач сегментации фона и детекции аномалий — **Farnebäck**.
